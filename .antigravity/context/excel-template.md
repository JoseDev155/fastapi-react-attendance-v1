* Desde el frontend o el backend esto se debe traducir a los valores en la Base de Datos:

```pgsql
CREATE TABLE attendances (
	id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
	attendance_date DATE NOT NULL,
	status ENUM('PRESENT', 'ABSENT', 'LATE', 'JUSTIFIED', 'LEFT_EARLY') NOT NULL, -- Ejemplo: 'Present', 'Absent', 'Late'
	notes VARCHAR(255),
	enrollment_id INTEGER REFERENCES enrollments(id) NOT NULL,
	UNIQUE(enrollment_id, attendance_date) -- Para evitar registros duplicados
);
```

* El `enrollment_id` deberás obtenerlo en base al apodo `nickname` o nombre del estudiante `first_name, last_name`.
* `notes` debe obtenerse creando comentarios en las celdas de asistencia, al escribir por ejemplo, TARDE, AUSENTE, JUSTIFICADO o SALIO TEMPRANO, debe tener la opción de agregar notas en la misma celda, y en el backend, extraer ese comentario y guardarlo en `notes`.
* **EXTRA:** El Excel que generes **debe** incluir una columna oculta (o de solo lectura) con el **ID único del estudiante** o la  **Matrícula** . Así, el backend no tiene que "adivinar" quién es quién; simplemente busca el ID.
